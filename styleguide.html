StyleGuide (UI components)

The following guide is designed to address developers handling the Front End/UI. The format is presented as [TITLE] [NOTES] [IMAGE/LIVE SAMPLE] [SNIPPET]. When you hover over a line, a green box will appear in the upper right. This allows you to open the snippet in a new window, copy to clipboard, or print it.

Elements are hilighted to identify attributes, values, etc. They are also line-numbered for easier long-term reference.

DISCLAIMER: This is a living document, and should be updated as things migrate to production. All items are taken from the development environment and may change.

Mixed: mobile and desktop

The Bootstrap 3 grid system has four tiers of classes: xs (phones), sm (tablets), md (desktops), and lg (larger desktops). You can use nearly any combination of these classes to create more dynamic and flexible layouts. Each tier of classes scales up, meaning if you plan on setting the same widths for xs and sm, you only need to specify xs.

Any element that is going to be used in a responsive form should have the following class form-control. This will allow the element to become block-level and exist on a line all by itself unless it gets wrapped with another div.
